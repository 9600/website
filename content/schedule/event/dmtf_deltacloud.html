The DMTF CIMI specification describes the model and protocol for management interactions between a cloud Infrastructure as a Service (IaaS) provider and the consumers of an IaaS service. The basic entities of IaaS (machines, storage, and networks) are modeled with the goal of providing consumer management access to an implementation of IaaS and facilitating portability between cloud implementations that support the specification. This talk will introduce the current work-in-progress CIMI specification and the initial implementation of that spec. as part of the Apache Deltacloud project.

Deltacloud exposes the CIMI API to HTTP clients and then facilitates management of IaaS resources across a number of heterogeneous cloud providers, even when those providers are non-conformant with respect to the CIMI specification.

The DMTF Cloud Management Working Group is a collaboration between a number of prominent participants, including Red Hat, VMware Inc., Oracle, IBM, Microsoft Corporation, Huawei, Fujitsu, Dell, Telefonica, Telecom Italia, France Telecom Group and others.
